> Think before you build. Plan before you code. Verify before you ship.

## The Workflow

```
PLAN → THINK → BUILD → REVIEW
```

1. **Plan** — Use `/plan-w-quizmaster` to understand what you're building. Answer the questions. Let the methodology force clarity before you touch code.

2. **Think** — Spawn polymathic agents to analyze your approach. Carmack for performance. Feynman for first principles. Jobs for product taste. Munger for risk.

3. **Build** — Use `/max-swarm` or `/max-parallel` to decompose the work and execute in parallel. One agent per file. No merge conflicts.

4. **Review** — Use `/consult-polymaths` to get multi-perspective review. Three different lenses on the same code.
